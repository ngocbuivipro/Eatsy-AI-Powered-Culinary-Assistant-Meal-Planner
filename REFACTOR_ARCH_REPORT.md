## 2. Key Changes & Technical Details

### A. Backend: Layered Architecture (Rule 2.B & 5.B)
| Component | Before | After | Impact |
| :--- | :--- | :--- | :--- |
| **Controllers** | Contained raw business logic, database queries, and third-party API calls. | Act only as entry points. Call Services and return standardized DTOs. | **Decoupling**: Easier to test and swap database/logic without breaking the API. |
| **Service Layer** | Non-existent or scattered. | Dedicated `*.service.js` files for 100% of business logic. | **Reusability**: Logic can be shared across multiple routes or even CLI tasks. |
| **DTO (Data Transfer Object)** | Returned raw Mongoose documents (including passwords/hashes). | Strict `*.dto.js` filters to clean up response data. | **Security**: Zero risk of leaking sensitive fields (e.g., `password`, `googleId`). |

**Modules refactored:** `User`, `Recipe`, `AI-Assistant`, `Pantry`.

### B. Anti-Hardcode Mandate (Rule 4)
We centralized all configuration and UI metadata to ensure the code remains "logic-pure."

1.  **System Config (`backend/src/config/index.js`)**: 
    - Centralized `process.env` access.
    - Added mandatory validation: Server will **Throw Error** immediately if vital API keys (Gemini, Spoonacular) are missing.
2.  **UI Colors (`frontend/src/constants/Colors.js`)**:
    - Sync'ed with `tailwind.config.js`.
    - Removed ~50+ hardcoded hex codes across navigators and screens.
3.  **UI Strings (`frontend/src/constants/Strings.js`)**:
    - Centralized all labels, placeholders, and error messages.
    - Enabled easy localization (Multi-language support) in the future.
4.  **API Endpoints (`frontend/src/constants/Endpoints.js`)**:
    - Centralized all routes. No more hardcoded `/api/v1/auth/login` strings in components.

### C. Security & Error Handling
- **Global Async Handler**: Standardized `catchAsync` used in all backend routes to eliminate `try-catch` boilerplate.
- **RESTful Responses**: Standardized `{ success, data, message }` format across all endpoints.
- **Environment Safety**: Guaranteed that internal DB structure changes won't affect the Frontend thanks to the DTO layer.
